# 📈 Ежедневный расчёт метрик пользователей (Airflow DAG)

Этот проект реализует **DAG в Apache Airflow**, который ежедневно в 12:00 UTC собирает и агрегирует данные о действиях пользователей за предыдущий день, а затем сохраняет результаты в ClickHouse.

---

## 🧠 Постановка задачи

Требуется автоматизировать расчёт ключевых метрик активности пользователей в соцсети. Данные поступают в две таблицы ClickHouse:

- **feed_actions** – просмотры и лайки контента  
- **message_actions** – отправленные и полученные сообщения  

Ожидается, что DAG будет:

1. **Параллельно** читать данные из обеих таблиц за вчерашний день (отдельные таски).
2. **Объединять** результаты в одну таблицу через `user_id` и дату.
3. **Агрегировать** метрики по трём срезам: полу (`gender`), операционной системе (`os`) и возрасту (`age`). Для каждого среза – своя таска.
4. **Сохранять** итоговые данные в отдельную таблицу ClickHouse, дополняя её новыми строками каждый день.

---

## 📁 Структура финальной таблицы (`test.danilov_airflow`)

| Колонка            | Тип      | Описание                                          |
|--------------------|----------|---------------------------------------------------|
| `event_date`       | Date     | Дата за которую посчитаны метрики (вчера)         |
| `dimension`        | String   | Название среза: `gender`, `os` или `age`          |
| `dimension_value`  | String   | Значение среза (например, `Male`, `iOS`, `25`)    |
| `views`            | UInt64   | Суммарное число просмотров                        |
| `likes`            | UInt64   | Суммарное число лайков                            |
| `messages_received`| UInt64   | Всего полученных сообщений                        |
| `messages_sent`    | UInt64   | Всего отправленных сообщений                      |
| `users_received`   | UInt64   | Количество уникальных отправителей, кто писал юзеру |
| `users_sent`       | UInt64   | Количество уникальных получателей, кому писал юзер |
